安装适用于 Linux 的 Windows 子系统

发表于 2021-09-18

Windows 10

对于 Windows 10，有两种方式安装适用于 Linux 的 Windows 子系统：

  • 手动安装：可以在任意版本的 Windows 10 上安装，但是需要执行多个步骤。
  • 简化安装：使用管理员权限执行命令 wsl --install 即可，但是需要加入 Windows 预览体验计划，并安装 Windows 10 的预览版（内部版本 20262 或更高版本）。

上述两种方式同样适用于 Windows 11；所有命令均在 Power Shell 中执行。

手动安装

启用 适用于 Linux 的 Windows 子系统

在 Windows 功能的窗口中，启用 适用于 Linux 的 Windows 子系统 功能，然后重启计算机。

或者，使用管理员权限执行命令：

更新到 WSL2（可选）

检查系统版本

WSL2 要求操作系统的最低版本为：

  • 对于 x64 系统，需要 1903（内部版本 18362）或更高版本。
  • 对于 ARM64 系统，需要 2004（内部版本 19041）或更高版本。

若要检查系统版本及内部版本号，按下 windows + R 键，然后输入 winver，点击“确定”即可。

启用 虚拟机平台

在 Windows 功能的窗口中，启用 虚拟机平台 功能，然后重启计算机。

或者，使用管理员权限执行命令：

下载 Linux 内核更新包
  • 适用于 x64 系统的 WSL2 Linux 内核更新包
  • 适用于 ARM64 系统的 WSL2 Linux 内核更新包

双击以运行下载的更新包。（系统将提示你提供提升的权限，选择“是”以批准此安装）

将 WSL2 设置为默认版本

安装 Linux 发行版

通过 Microsoft Store 安装

打开 Microsoft Store，搜索 linux，按喜好安装。

直接安装

在许多情况下，你可能无法（或不想）通过 Microsoft Store 安装 WSL Linux 发行版。具体而言，你可能正在运行不支持 Microsoft Store 的长期服务系统 (Windows LTSC) ，或者你的公司网络策略 和/或 管理员不允许在你的环境中使用 Microsoft Store。

如果无法使用 Microsoft Store，可以点击下面的链接下载并手动安装 Linux 发行版：

  • Ubuntu 20.04
  • Ubuntu 20.04 ARM
  • Ubuntu 18.04
  • Ubuntu 18.04 ARM
  • Ubuntu 16.04
  • Debian GNU/Linux
  • Kali Linux
  • SUSE Linux Enterprise Server 12
  • SUSE Linux Enterprise Server 15 SP2
  • OpenSUSE Leap 15.2
  • Fedora Remix for WSL

配置

首次启动新安装的 Linux 发行版时，将打开一个控制台窗口，系统会要求你等待一分钟或两分钟，以便文件解压缩并存储到电脑上。 未来的所有启动时间应不到一秒。

然后，需要为新的 Linux 发行版创建用户帐户和密码。

简化安装

在最新的 Windows 预览体验预览版中，显著改进了适用于 Linux 的 Windows 子系统的安装过程，从原本的多个步骤替换为一条命令。
使用简化安装，必须加入 Windows 预览体验计划 并安装 Windows 10 的预览版（20262 或更高版本），然后使用管理员权限执行命令：

默认情况下，安装的 Linux 发行版为 Ubuntu。可以使用 wsl --install -d <Distribution Name> 进行更改。（将 <Distribution Name> 替换为所需发行版的名称）若要查看可用 Linux 发行版的列表，请输入 wsl –list –online。

初始安装后，可以使用 wsl --install -d <Distribution Name> 命令将其他 Linux 发行版添加到计算机。
